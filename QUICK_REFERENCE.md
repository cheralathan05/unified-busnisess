## 🎯 API Endpoints

### Submit Intake
```
POST /api/client-intake/submit
Authorization: Bearer {token}

Request: { businessName, email, projectType, ... }
Response: { leadId, requirementId, submissionId, aiInsights }
```

### Get Submission
```
GET /api/client-intake/submission/:id
Response: { formData, aiSummary, completionScore, ... }
```

### List Submissions
```
GET /api/client-intake/submissions?limit=20&offset=0
Response: { data: [...], pagination: {...} }
```

### Analyze Submission
```
POST /api/client-intake/:id/analyze
Response: { completionScore, insights, risks, recommendations }
```

## 💾 Database Schema (Prisma)

```prisma
IntakeSubmission
  ├── id: String (cuid)
  ├── leadId: String (FK→Lead)
  ├── formData: Json (entire form)
  ├── aiSummary: String
  ├── completionScore: Int
  └── suggestions: String[]

Requirement
  ├── id: String (cuid)
  ├── title: String
  ├── description: String
  ├── leadId: String (FK→Lead)
  ├── features: String[]
  ├── scope: Json
  ├── status: String (default: draft)
  └── priority: String
```
